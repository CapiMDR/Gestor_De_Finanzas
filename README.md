# Gestor_De_Finanzas
hola mundo prueba
| Requerimiento Funcional (RF) | Requerimiento No Funcional (RNF) |
|------------------------------|---------------------------------|
| El sistema permite al usuario registrar y gestionar más de una cuenta financiera. | El sistema deberá mostrar el balance total del usuario y los gráficos financieros en menos de 5 segundos tras seleccionar un rango de fechas. |
| El sistema permite definir si una cuenta es física (efectivo) o digital (bancaria). | Las búsquedas y filtros de movimientos específicos deberán ejecutarse en menos de 5 segundos. |
| El usuario podrá asignar una moneda específica (ej. USD, MXN) a cada cuenta. | Las operaciones básicas (registro de ingresos/gastos, consultas de balance) deben estar siempre accesibles sin caídas prolongadas. |
| El sistema permite establecer un saldo de partida al crear una nueva cuenta. | El sistema debe garantizar que ninguna transacción se pierda o corrompa, incluso ante un cierre inesperado o fallo de energía. |
| El sistema permite al usuario visualizar los gastos realizados dentro de un rango de fechas seleccionado. | El software debe contar con una interfaz adaptativa e intuitiva, con un tiempo de aprendizaje por usuario no mayor a 2 horas. |
| Se le permite al usuario visualizar ingresos recibidos dentro de un rango de fechas seleccionado. | El software debe estar optimizado para funcionar en dispositivos con mínimo 4 GB de RAM en adelante. |
| Permite mostrar el total de las finanzas del usuario, calculando y desplegando el balance final (ingresos menos gastos). | El sistema operativo estará diseñado para dispositivos con Windows 10. |
| El software permite crear recordatorios o alertas generales dentro del sistema. | Los logs de errores deberán permitir identificar fallas y problemas rápidamente. |
| Permitirá borrar las notificaciones que el usuario ya no necesite. | El software no ocupará más de 100 MB en espacio de disco. |
| El sistema permitirá la creación de pagos habituales, es decir, registrar pagos recurrentes (ej. renta, suscripciones). | - |
| Se le permite al usuario borrar los pagos recurrentes que ya no sean válidos. | - |
| Se le proporciona al usuario una opción para marcar que un pago habitual ha sido realizado. | - |
| El sistema debe calcular el crecimiento de una cantidad de dinero a futuro con una tasa de interés. | - |
| Se le permite al usuario cambiar la apariencia visual del programa, como los colores. | - |
| El sistema permite categorizar un gasto usando una lista de opciones predefinidas. | - |
| Permitirá al usuario añadir categorías de gasto personalizadas a la lista. | - |
| Se le debe permitir al usuario borrar las categorías de gasto que él haya creado. | - |
| El sistema debe permitir registrar la fecha exacta en que se efectuó un gasto. | - |
| Permitirá añadir notas o descripciones a cada transacción de gasto. | - |
| Permitirá editar o borrar las notas de una transacción de gasto. | - |
| El sistema debe permitir categorizar un ingreso usando una lista de opciones predefinidas. | - |
| Debe permitir al usuario añadir y eliminar categorías de ingreso personalizadas a la lista. | - |
| Permitirá registrar la fecha exacta en que se recibió un ingreso. | - |
| Debe permitir añadir notas o descripciones a cada transacción de ingreso. | - |
| El sistema permite editar o borrar las notas de una transacción de ingreso. | - |
| El sistema es capaz de generar gráficos de barras y de pastel que representen los datos de un rango de fechas seleccionado. | - |
| Se le permite al usuario visualizar la distribución de datos financieros en formato de gráfico circular. | - |
| El sistema puede generar y mostrar al usuario un informe consolidado de las finanzas de forma semanal o mensual. | - |
| El sistema permite al usuario establecer metas de ahorro y monitorear el avance. | - |
| Se debe contar en el sistema con una herramienta para buscar y filtrar movimientos específicos. | - |
